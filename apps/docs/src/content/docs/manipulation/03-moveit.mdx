[MoveIt 2](https://moveit.picknik.ai/) est **la** stack ROS 2 de
planification de mouvement. Au lieu de calculer vous-même chaque
trajectoire articulaire, vous lui donnez un objectif (pose cartésienne,
configuration cible) et il vous renvoie une trajectoire collision-free.

## Ce que MoveIt apporte

- **Cinématique inverse** (IK) : « quelle config de joints donne cette
  pose de la pince ? » — résolu par KDL ou TracIK.
- **Planification** (OMPL, STOMP, Pilz…) : « comment passer de A à B sans
  rentrer dans un obstacle ? ».
- **Planning Scene** : représentation du monde (obstacles, objets attachés).
- **Trajectory execution** : envoie la trajectoire planifiée du bras au
  `arm_controller` (le `joint_trajectory_controller` vu au chapitre
  précédent) ; la pince est pilotée séparément par le `gripper_controller`
  (action `ParallelGripperCommand`).

## Architecture

```text
   RViz MotionPlanning panel
            │
            ▼
   move_group node ────┬──► OMPL / STOMP planners
            │          │
            │          └──► IK plugin (KDL / TracIK)
            │
            ▼
   /arm_controller/follow_joint_trajectory
            │                          (pince : /gripper_controller/gripper_cmd)
            ▼
        Robot (sim ou réel)
```

`move_group` est le node central. Il consomme la description du robot
(URDF + SRDF) et expose les services / actions de planification.

## Le SRDF — ce qu'on définit en plus de l'URDF

L'URDF décrit la géométrie. Pour MoveIt il faut **aussi** un **SRDF**
(Semantic Robot Description Format), généré via le **MoveIt Setup
Assistant**, qui ajoute :

- les **groupes** de joints (`arm`, `gripper`) ;
- les **poses nommées** (`home`, `tuck`, `pick_ready`) ;
- les **paires de collisions à ignorer** (calcul auto sur des poses
  aléatoires) ;
- les **end-effectors** (lien entre `arm` et `gripper`).

## Premier lancement

<Aside type="note" title="Hands-on (à venir)">
Quand la config sera prête :

```bash
ros2 launch so101_moveit_config demo.launch.py
```

Cela démarre `move_group` + RViz avec le **MotionPlanning panel**
préchargé. Dans RViz :

1. Onglet **Planning** → drag de la sphère interactive autour de la pince
   pour définir une cible cartésienne.
2. Bouton **Plan** : MoveIt cherche une trajectoire.
3. Bouton **Execute** : la trajectoire est jouée sur le bras
   (sim ou réel selon le launch).

</Aside>

## Planification cartésienne

Pour des trajectoires en ligne droite (utile pour les approches de pick),
MoveIt expose `compute_cartesian_path` (service `/compute_cartesian_path`
ou méthode Python `MoveItPy`). Vous lui donnez une liste de waypoints
cartésiens et vous obtenez une trajectoire interpolée.

## Collisions

Le **Planning Scene** est mis à jour en publiant des `CollisionObject`
sur le topic `/collision_object`. Exemple :

```python
from moveit_msgs.msg import CollisionObject
from shape_msgs.msg import SolidPrimitive
from geometry_msgs.msg import Pose

obj = CollisionObject()
obj.id = "table"
obj.header.frame_id = "base_link"

box = SolidPrimitive()
box.type = SolidPrimitive.BOX
box.dimensions = [1.0, 1.0, 0.05]

pose = Pose()
pose.position.x = 0.3
pose.position.z = -0.025

obj.primitives.append(box)
obj.primitive_poses.append(pose)
obj.operation = CollisionObject.ADD
```

MoveIt évitera automatiquement cet obstacle dans ses planifications.
