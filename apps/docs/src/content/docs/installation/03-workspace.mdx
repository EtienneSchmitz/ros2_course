Le workspace `~/ros2_bootcamp_ws/` est votre dossier de travail local pour tout le cours. **Il n'est pas versionné** : vous le construisez sur votre machine et vous y tirez les repos robots via [vcstool](https://github.com/dirk-thomas/vcstool).

## Installer les outils workspace

```bash
sudo apt install -y \
  python3-colcon-common-extensions \
  python3-rosdep \
  python3-vcstool \
  git
```

Initialisez `rosdep` (une seule fois sur la machine) :

```bash
sudo rosdep init
rosdep update
```

## Créer le dossier du workspace

```bash
mkdir -p ~/ros2_bootcamp_ws/src
cd ~/ros2_bootcamp_ws
```

Structure cible :

```text
~/ros2_bootcamp_ws/
├── bootcamp.repos     # liste des repos robots à cloner (téléchargée ci-dessous)
└── src/               # vide au départ, peuplé par vcstool
```

<Aside type="caution">
Ne faites **pas** de `git init` sur `~/ros2_bootcamp_ws/`. Ce dossier reste local. Le code source des robots, lui, vivra dans des repos GitHub séparés clonés dans `src/`.
</Aside>

## Télécharger `bootcamp.repos`

Le fichier `bootcamp.repos` liste les repos robots à cloner. Il est hébergé sur le site du cours :

```bash
cd ~/ros2_bootcamp_ws
wget https://ros2.etienne-schmitz.com/bootcamp.repos
```

## Tirer les repos avec vcstool

```bash
vcs import src < bootcamp.repos
```

<Aside type="note">
Au démarrage du cours, `bootcamp.repos` est **vide** (les repos robots arrivent au fur et à mesure). La commande ci-dessus s'exécute sans erreur mais ne clone rien — c'est normal. Vous rejouerez `vcs import` plus tard pour récupérer les robots dès qu'ils seront publiés.
</Aside>

## Installer les dépendances ROS

```bash
cd ~/ros2_bootcamp_ws
rosdep install --from-paths src --ignore-src -r -y
```

Quand `src/` est vide, la commande passe en silence ; quand des packages y vivent, elle installe leurs dépendances système.

## Premier `colcon build`

```bash
colcon build --symlink-install
```

Le flag `--symlink-install` permet de modifier les fichiers Python / launch sans rebuilder.

## Sourcer ROS et le workspace

À chaque ouverture de terminal, **deux** choses doivent être sourcées : d'abord
**ROS 2**, puis votre **workspace**. La source ROS a déjà été ajoutée à votre
`~/.bashrc` à l'[étape précédente](/installation/02-ros-gazebo/) ; il ne reste
qu'à y ajouter le workspace :

```bash
echo "source ~/ros2_bootcamp_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

Votre `~/.bashrc` doit alors contenir les **deux** lignes, dans cet ordre :

```bash
source /opt/ros/kilted/setup.bash             # ajouté à l'étape ROS
source ~/ros2_bootcamp_ws/install/setup.bash  # ajouté ici
```

<Aside type="caution" title="À savoir sur le sourcing automatique">
Mettre `install/setup.bash` dans le `.bashrc` est pratique, mais comporte des
effets de bord à connaître :

- **Tout nouveau terminal** hérite de l'overlay du workspace. Si un `colcon build`
  échoue ou laisse le workspace dans un état cassé, vos terminaux peuvent afficher
  des erreurs dès l'ouverture.
- Après un build qui **ajoute un nouveau package**, ouvrez un **terminal neuf**
  (ou re-sourcez) pour qu'il soit visible.
- Si vous jonglez avec **plusieurs workspaces**, l'auto-source fige celui-ci :
  préférez alors l'alias ci-dessous et sourcez à la demande.

```bash
alias rosws='source ~/ros2_bootcamp_ws/install/setup.bash'
```
</Aside>

<Aside type="tip">
La suite du cours **suppose** que ROS et le workspace sont sourcés
automatiquement via votre `~/.bashrc`. Les parties ne répètent donc **pas** ces deux
`source` dans leurs blocs de commandes.
</Aside>
