Le workspace `~/ros2_bootcamp_ws/` est ton dossier de travail local pour tout le cours. **Il n'est pas versionné** : tu le construis sur ta machine et tu y tires les repos robots via [vcstool](https://github.com/dirk-thomas/vcstool).

## Installer les outils workspace

```bash
sudo apt install -y \
  python3-colcon-common-extensions \
  python3-rosdep \
  python3-vcstool \
  git
```

Initialise `rosdep` (une seule fois sur la machine) :

```bash
sudo rosdep init
rosdep update
```

## Créer le dossier du workspace

```bash
mkdir -p ~/ros2_bootcamp_ws/src
cd ~/ros2_bootcamp_ws
```

Structure cible :

```text
~/ros2_bootcamp_ws/
├── bootcamp.repos     # liste des repos robots à cloner (téléchargée ci-dessous)
└── src/               # vide au départ, peuplé par vcstool
```

<Aside type="caution">
Ne fais **pas** de `git init` sur `~/ros2_bootcamp_ws/`. Ce dossier reste local. Le code source des robots, lui, vivra dans des repos GitHub séparés clonés dans `src/`.
</Aside>

## Télécharger `bootcamp.repos`

Le fichier `bootcamp.repos` liste les repos robots à cloner. Il est hébergé sur le site du cours :

```bash
cd ~/ros2_bootcamp_ws
wget https://ros2.etienne-schmitz.com/bootcamp.repos
```

## Tirer les repos avec vcstool

```bash
vcs import src < bootcamp.repos
```

<Aside type="note">
Au démarrage du cours, `bootcamp.repos` est **vide** (les repos robots arrivent au fur et à mesure). La commande ci-dessus s'exécute sans erreur mais ne clone rien — c'est normal. Tu rejoueras `vcs import` plus tard pour récupérer les robots dès qu'ils seront publiés.
</Aside>

## Installer les dépendances ROS

```bash
cd ~/ros2_bootcamp_ws
rosdep install --from-paths src --ignore-src -r -y
```

Quand `src/` est vide, la commande passe en silence ; quand des packages y vivent, elle installe leurs dépendances système.

## Premier `colcon build`

```bash
colcon build --symlink-install
```

Le flag `--symlink-install` permet de modifier les fichiers Python / launch sans rebuilder.

## Sourcer le workspace

À chaque ouverture de terminal, tu dois sourcer **deux choses** : ROS (déjà fait via `~/.bashrc`) puis le workspace :

```bash
source ~/ros2_bootcamp_ws/install/setup.bash
```

Pour automatiser, ajoute à ton `~/.bashrc` :

```bash
echo "source ~/ros2_bootcamp_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

Ou un alias plus contrôlé :

```bash
alias rosws='source ~/ros2_bootcamp_ws/install/setup.bash'
```
