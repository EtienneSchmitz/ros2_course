ROS 2 et Gazebo s'installent ensemble : le bridge `ros_gz` fait le lien entre les deux et c'est la combinaison utilisée tout au long du cours.

## Locale UTF-8

ROS 2 nécessite une locale UTF-8 :

```bash
locale  # vérifie la locale courante

sudo apt update && sudo apt install -y locales
sudo locale-gen fr_FR fr_FR.UTF-8
sudo update-locale LC_ALL=fr_FR.UTF-8 LANG=fr_FR.UTF-8
export LANG=fr_FR.UTF-8
```

## Ajouter le dépôt ROS 2

```bash
sudo apt install -y software-properties-common
sudo add-apt-repository universe

sudo apt update && sudo apt install -y curl
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
  | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
```

## Installer ROS 2 Kilted

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y ros-kilted-desktop python3-argcomplete
```

Le métapaquet `ros-kilted-desktop` inclut RViz 2, les démos, et la majorité des paquets utiles.

Quelques utilitaires ne sont **pas** tirés par `desktop` mais sont utilisés tout au long du cours (visualisation URDF, sliders de joints) :

```bash
sudo apt install -y \
  ros-kilted-joint-state-publisher \
  ros-kilted-joint-state-publisher-gui
```

## Sourcer ROS 2

Ajoutez le sourcing à votre `~/.bashrc` pour ne pas avoir à le refaire à chaque terminal :

```bash
echo "source /opt/ros/kilted/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

Vérifiez que `ros2` est trouvé :

```bash
ros2 --help
```

## Tester ROS 2 (talker / listener)

Dans **deux terminaux** différents :

<Tabs>
<TabItem label="Terminal 1 — talker">

```bash
ros2 run demo_nodes_cpp talker
```

</TabItem>
<TabItem label="Terminal 2 — listener">

```bash
ros2 run demo_nodes_py listener
```

</TabItem>
</Tabs>

Le listener doit afficher en continu les messages `Hello World: N` publiés par le talker. Si oui, ROS 2 est opérationnel.

## Installer Gazebo Harmonic

Gazebo Harmonic vient des dépôts OSRF :

```bash
sudo curl -sSL https://packages.osrfoundation.org/gazebo.gpg \
  -o /usr/share/keyrings/pkgs-osrf-archive-keyring.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/pkgs-osrf-archive-keyring.gpg] \
http://packages.osrfoundation.org/gazebo/ubuntu-stable $(lsb_release -cs) main" \
  | sudo tee /etc/apt/sources.list.d/gazebo-stable.list > /dev/null

sudo apt update && sudo apt install -y gz-harmonic
```

Tester Gazebo seul :

```bash
gz sim shapes.sdf
```

Une fenêtre Gazebo doit s'ouvrir avec quelques formes. Fermez-la (`Ctrl+C` dans le terminal).

## Installer le bridge `ros_gz`

Le bridge `ros_gz` permet à ROS 2 et Gazebo de s'échanger des messages :

```bash
sudo apt install -y ros-kilted-ros-gz
```

Test rapide, dans **trois terminaux** :

<Tabs>
<TabItem label="Terminal 1 — Gazebo">

```bash
gz sim -r shapes.sdf
```

</TabItem>
<TabItem label="Terminal 2 — bridge">

```bash
ros2 run ros_gz_bridge parameter_bridge \
  /clock@rosgraph_msgs/msg/Clock@gz.msgs.Clock
```

</TabItem>
<TabItem label="Terminal 3 — echo /clock">

```bash
ros2 topic echo /clock
```

</TabItem>
</Tabs>

Vous devez voir le `/clock` ROS avancer pendant que Gazebo simule.
