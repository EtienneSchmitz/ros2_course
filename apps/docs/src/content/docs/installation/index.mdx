L'objectif : une machine prête avec **Ubuntu 24.04**, **ROS 2 Kilted**, **Gazebo Ionic** et le workspace du cours. Deux parcours au choix, suivis d'une vérification commune.

## Pré-requis matériels

- Ordinateur capable de virtualiser une scène Gazebo Ionic (recommandé : **8 Go de RAM**, **CPU 4 cœurs**, **20 Go libres** sur disque).
- GPU dédié optionnel mais recommandé pour Gazebo et RViz fluides.
- Connexion Internet pour les téléchargements de paquets et de repos.

<Aside type="tip" title="Quel parcours selon votre OS ?">
- **Linux** → installation native.
- **Windows** → **WSL2** pour la simulation ; **dual-boot natif** pour piloter les vrais robots.
- **macOS** → Docker.
</Aside>

Une fois l'un des deux parcours terminé, passez à la **[vérification](/installation/03-verification/)** (`ros2 doctor`, build à vide, ouverture de Gazebo) — commune aux deux.

### Sur Windows : WSL2

Pour la **simulation**, le plus simple sous Windows est [WSL2](https://learn.microsoft.com/windows/wsl/), qui fait tourner un vrai Ubuntu 24.04 dans Windows (Gazebo et RViz s'affichent via WSLg). Dans un PowerShell **administrateur** :

```powershell
wsl --install -d Ubuntu-24.04
```

Ouvrez ensuite le terminal Ubuntu et suivez l'[installation native](/installation/01-ubuntu/) : tout s'applique à l'identique (sautez la section « Installer Ubuntu 24.04 »).
