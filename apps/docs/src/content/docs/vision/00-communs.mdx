## 3. `cv_bridge` : du message ROS au tableau OpenCV

Un nœud ROS reçoit des `sensor_msgs/Image`. `cv_bridge` les convertit en image OpenCV
(un tableau NumPy `H × W × C`, en **BGR**) :

```python
from cv_bridge import CvBridge
bridge = CvBridge()
frame = bridge.imgmsg_to_cv2(msg, desired_encoding="bgr8")   # -> np.ndarray BGR
```

## 5. Estimer la pose : back-projection sur le plan de la table

`Detection.msg` attend une `geometry_msgs/Pose` (3D). Avec une caméra **en vue de
dessus** et les intrinsèques connues, on remonte d'un pixel `(u, v)` vers un point 3D
sur le plan de la table. La distance caméra→table est documentée dans le **README de
`bootcamp_vision`**.

```python
# Intrinsèques de la caméra bootcamp_vision (cf. /camera/camera_info)
FX, FY, CX, CY = 935.485, 935.485, 640.0, 360.0
CAM_HEIGHT = 0.80   # distance caméra → plan de la table (m) — voir README bootcamp_vision

def backproject(u, v):
    """Pixel (u,v) -> (x, y, z) sur le plan de la table (repère caméra)."""
    x = (u - CX) / FX * CAM_HEIGHT
    y = (v - CY) / FY * CAM_HEIGHT
    return x, y, 0.0
```

<Aside type="caution" title="Pose 2D dégradée">
Cette pose suppose l'objet **posé à plat** (z connu) et ignore son orientation
(quaternion identité). C'est suffisant pour le pick & place, mais ce n'est **pas** une
vraie pose 6D. Seul l'atelier [ArUco](/vision/02-aruco/) fournit une pose **6D
complète** — c'est pourquoi ArUco est le détecteur le plus fiable.
</Aside>

## 6. Le squelette du nœud `detector`

Voici le **patron commun** : il souscrit à la caméra, convertit l'image, et publie sur
`/detections`. Chaque atelier ne remplace que le bloc « détection » dans `on_image`.

```python
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from geometry_msgs.msg import Pose
from cv_bridge import CvBridge

from mon_projet_interfaces.msg import Detection   # ⚠️ votre package d'interfaces (J1)

FX, FY, CX, CY = 935.485, 935.485, 640.0, 360.0
CAM_HEIGHT = 0.80


class Detector(Node):
    def __init__(self):
        super().__init__("detector")
        self.bridge = CvBridge()
        self.pub = self.create_publisher(Detection, "/detections", 10)
        self.create_subscription(Image, "/camera/image_raw", self.on_image, 10)
        self.get_logger().info("detector prêt — j'écoute /camera/image_raw")

    def on_image(self, msg):
        frame = self.bridge.imgmsg_to_cv2(msg, "bgr8")
        # >>> bloc spécifique à l'atelier : produire (class_id, u, v) <<<
        # x, y, z = self.backproject(u, v)
        # self.publier(class_id, x, y, z)

    def backproject(self, u, v):
        return (u - CX) / FX * CAM_HEIGHT, (v - CY) / FY * CAM_HEIGHT, 0.0

    def publier(self, class_id, x, y, z, quat=(0.0, 0.0, 0.0, 1.0)):
        det = Detection()
        det.class_id = class_id
        det.pose = Pose()
        det.pose.position.x, det.pose.position.y, det.pose.position.z = x, y, z
        (det.pose.orientation.x, det.pose.orientation.y,
         det.pose.orientation.z, det.pose.orientation.w) = quat
        self.pub.publish(det)


def main():
    rclpy.init()
    rclpy.spin(Detector())
    rclpy.shutdown()
```

Pour tester n'importe quel atelier, avec le monde lancé et un objet spawné :

```bash
ros2 run mon_projet_vision detector     # votre package
ros2 topic echo /detections
```
